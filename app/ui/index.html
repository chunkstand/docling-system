System diagram

Two loops share one contract: durable state first, live promotion second.

Input PDF ingest

Upload or local-file CLI queues a durable `document_run`.

Persistence Versioned run artifacts

Chunks, tables, figures, `docling.json`, YAML, and run metadata are stored.

Gate Validation before activation

The active corpus only advances when document and table validation pass.

Serving Search and grounded chat

Operators query only the last validated active run.

Oversight Replay, eval, and telemetry

Search history, replay suites, evaluation corpora, and operator labels stay durable.

Agent layer Draft, verify, approve

Agent tasks triage regressions, draft harnesses, verify them, and wait for approval.

Promotion discipline Gated change publication

Agents can recommend and prepare changes; humans approve any promotable step.

Validation-gated promotion

Broken reprocesses never replace the live corpus.

`documents.active_run_id` only advances after validation passes. Failed runs remain non-active and searchable content stays stable.

Observability

Artifacts, outcomes, verifier rows, traces, and replay deltas stay inspectable.

The system persists enough evidence to explain why a run, recommendation, or harness change was considered safe or unsafe.

Governed retrieval

Named harnesses make retrieval behavior reviewable and comparable.

Operators can compare default, wide, prose, and applied review harnesses against the same replay sources before any promotion.

Agent orchestration

Agents improve the system through bounded workflows, not hidden side effects.

Triage, drafting, verification, approval, and rejection all move through durable task records with explicit lineage.

What the system does

It is a retrieval platform and a governed improvement loop.

Ingest

PDFs become versioned document runs with normalized storage and durable artifacts.

Validate

Document and table validations decide whether a run may become active.

Retrieve

Search returns mixed chunk and table evidence under named harnesses.

Evaluate

Replay suites and evaluation corpora measure retrieval quality over time.

Improve

Agent tasks draft bounded changes, verify them, and wait for explicit approval.
